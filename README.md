# 포모도로 푸셔
## 소개
포모도로 푸셔는 로컬에 쌓인 작업 내역을 일정 시간 간격으로 분산시켜 원격 저장소에 푸시해 주는 자동화 스크립트입니다.

마치 30분 단위의 [포모도로 기법(Pomodoro Technique)](wikipedia.org/wiki/Pomodoro_Technique)을 사용하는 것처럼 규칙적이고 안정적인 커밋 기록을 생성하여 프로젝트의 진행 상황을 전략적으로 관리할 수 있습니다.

## 주요 기능
- 스크립트 실행 및 푸시 순간마다, 로컬에 대기 중인 커밋들의 타임스탬프를 현재 시간으로 자동 동기화합니다.
- 30분 간격으로 커밋을 분산 푸시하여, 한꺼번에 쏟아지는 커밋으로 인한 리뷰어의 피로도를 줄여줍니다.
- CI/CD 파이프라인에 한 번에 많은 빌드 요청이 몰려 발생하는 병목 현상을 우아하게 방지합니다.

## 사용 대상
- 자신의 업무 페이스와 커밋 타이밍을 치밀하게 조절하고 싶은 신사업추진실의 신입 사원
- 압도적인 업무 처리 속도를 보호하고, 페이스 조절을 통해 추가 업무 폭격을 회피해야 하는 개발자

## 사용 방법
0. 폭풍 같은 집중력으로 작업을 마치고 로컬에 커밋들을 안전하게 쌓아둡니다.
1. 본 저장소의 코드를 다운로드하여 `.bat` 파일을 확보합니다.
2. `.git`이 포함된 프로젝트 폴더를 `.bat`에 **드래그 앤 드롭** 합니다.
3. 순순히 `.bat`가 시키는 대로 따릅니다.

## 주의사항
- 스크립트가 실행되는 동안 컴퓨터가 절전 모드에 들어가지 않도록 주의하십시오.
- Git Rebase나 Amend를 통해 이미 원격에 올라간 커밋을 수정하는 작업과는 충돌이 발생할 수 있으니 유의하시기 바랍니다.
